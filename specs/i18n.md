# Internationalisation (i18n)

**Files:**
- `src/i18n/types.ts` — `Translations` type + `TFunction` type
- `src/i18n/en.ts` — English strings (source of truth)
- `src/i18n/zh.ts` — Chinese (Simplified)
- `src/i18n/es.ts` — Spanish
- `src/i18n/ru.ts` — Russian
- `src/i18n/index.ts` — React context, hooks, helpers
- `src/components/LanguageSwitcher.tsx` — globe icon + dropdown
- `api/translate.ts` — Vercel serverless: on-demand OpenAI translation

---

## Architecture

### Provider setup

Wrap the app root in `I18nProvider`:
```tsx
// App.tsx
import { I18nProvider } from './i18n'
export default function App() {
  return <I18nProvider><YourScreen /></I18nProvider>
}
```

### In React components

```ts
import { useT } from "../i18n";
const t = useT();
t("game.correct")                          // → "Correct! 🎉"
t("game.wrongAnswer", { answer: 5 })       // → "The answer was 5"
```

### Outside React (PDF generation, share report)

```ts
import { getT } from "../i18n";
const t = getT(locale);   // locale is a string like "en", "zh"
```

`getT` reads from `localStorage` for cached custom translations.

### Locale persistence

Active locale is stored as `localStorage["lang"]` (key `"lang"`).
Initialized from `localStorage` on mount, defaults to `"en"`.

---

## `TFunction` type

```ts
export type TFunction = (key: TranslationKey, vars?: Record<string, string | number>) => string;
```

Interpolates `{placeholder}` tokens. Unknown keys fall back to English (`en[key] ?? key`).

## `src/i18n/index.ts` — full API

### Context

```ts
interface I18nContextValue {
  locale: string;
  setLocale: (code: string) => void;
  t: TFunction;
}
```

### Exported hooks

```ts
export function useT(): TFunction
// Returns the translation function for the current locale.

export function useLocale(): { locale: string; setLocale: (code: string) => void }
// Returns the current locale code and setter.
```

### Non-hook helpers

```ts
export function getT(locale: string): TFunction
// Build a translation function for the given locale (for non-React code).
// Falls back to English for missing keys.

export function cacheTranslation(locale: string, translations: Translations): void
// Save a custom translation to localStorage (key: "i18n_cache_<locale>").

export function saveCustomLang(code: string, name: string): void
// Persist a custom language name mapping in localStorage (key: "i18n_custom_langs").

export function getCustomLangs(): Record<string, string>
// Load all previously translated custom language names from localStorage.
```

### Built-in locales

```ts
export const BUILT_IN_LOCALES: Record<string, Translations> = { en, zh, es, ru, hi };

export const LOCALE_NAMES: Record<string, string> = {
  en: "English", zh: "中文", es: "Español", ru: "Русский", hi: "हिन्दी",
};
```

### Interpolation

`{placeholder}` tokens in translation strings are replaced by values from `vars`:
```ts
t("game.tapScreen", { count: 2, total: 5 })  // → "Tap 2/5"
```
Unknown placeholders are left as-is: `"{unknown}"`.

## `LanguageSwitcher` component

Globe icon (🌐) button in the top-right toolbar cluster.

On click → opens a dropdown with:
- One button per built-in language (EN, ZH, ES, RU, HI)
- Any previously translated custom languages (from `getCustomLangs()`)
- "Other..." option to trigger on-demand translation

**On-demand translation flow:**
1. User clicks "Other..." → dialog asks for language name
2. `POST /api/translate` with `{ targetLang: "French", strings: en }` (all English strings)
3. Response: `{ translations: {...}, langCode: "fr" }`
4. `cacheTranslation("fr", translations)` + `saveCustomLang("fr", "Français")`
5. `setLocale("fr")` — dropdown now includes "Français"

## On-demand translation API (`api/translate.ts`)

**Endpoint:** `POST /api/translate`

**Required env var:** `OPENAI_API_KEY`

**Request body:**
```ts
{ targetLang: string, strings: Record<string, string> }
```

**Response:**
```ts
{ translations: Record<string, string>, langCode: string }
```

**Model:** `gpt-4o-mini`, temperature 0.3, `response_format: { type: "json_object" }`

**System prompt rules:**
1. Preserve all `{placeholder}` tokens exactly
2. Do not translate URLs
3. Do not translate brand names: SeeMaths, Ripple Touch, DiscussIt, Interactive Maths
4. Return `{ "translations": {...}, "langCode": "<ISO 639-1>" }`

## Adding new string keys (for new games)

1. Add the key to `src/i18n/types.ts` `Translations` type
2. Add the English value to `src/i18n/en.ts`
3. Add translations to `src/i18n/zh.ts`, `es.ts`, `ru.ts`
4. TypeScript will error if any locale file is missing the key

## PDF and email language support

`generateSessionPdf(summary, t)` accepts `t` so the PDF is generated in whatever
locale is active. Same for `buildEmailStrings(summary, t)` in `shareReport.ts`.

jsPDF uses Helvetica (Latin), which handles CJK and Cyrillic if the system font
substitutes. For guaranteed CJK/Cyrillic rendering, embed fonts via
`doc.addFont()` + `doc.setFont()` before drawing text.
