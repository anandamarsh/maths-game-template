## Feature: Internationalisation (i18n)

**Playbook:** `i18n.md` (detailed step-by-step playbook)
**Implementation:** `src/i18n/`, `src/components/LanguageSwitcher.tsx`, `api/translate.ts`

### What it does
All user-visible strings are externalised into locale files. The game ships with
four built-in languages (English, Chinese, Spanish, Russian). Additional languages
can be translated on demand via OpenAI and cached in localStorage.

### Built-in Languages
| Code | Language | Script |
|------|----------|--------|
| `en` | English | Latin |
| `zh` | Chinese (Simplified) | Han |
| `es` | Spanish | Latin |
| `ru` | Russian | Cyrillic |

### UI
Globe icon in the toolbar opens a language dropdown. Selection is instant and
persisted across sessions via localStorage.

### Architecture
- `src/i18n/index.ts` - React context + `useT()` hook + `getT()` for non-React
- `src/i18n/en.ts` - English strings (source of truth)
- `src/i18n/zh.ts`, `es.ts`, `ru.ts` - Static translations
- `src/components/LanguageSwitcher.tsx` - Globe icon + dropdown
- `api/translate.ts` - Vercel serverless function for on-demand translation

### PDF & Email
Reports are generated in the selected language. The email body and PDF text
both respect the active locale. jsPDF uses embedded fonts for CJK and Cyrillic
support.

### On-demand Translation
Selecting "Other..." in the language dropdown prompts for a language name, calls
the OpenAI translation API, and caches the result in localStorage for future use.

### Integration requirements for new games
1. Import `useT` from `src/i18n` in all components with user-visible text.
2. Replace hardcoded strings with `t("key.name")` or `t("key.name", { var })`.
3. For non-React code (e.g. PDF generation), accept a `t` function parameter.
4. Add any new string keys to `src/i18n/en.ts` and all locale files.
